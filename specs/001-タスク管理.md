# タスク管理機能

## 概要
シンプルで直感的なタスク管理アプリケーション。ユーザーが日常のタスクを追加・管理し、進捗を把握できるようにする。SPA（Single Page Application）としてブラウザ上で動作し、データはlocalStorageに永続化する。

## 技術スタック
- HTML / CSS / JavaScript（バニラ、フレームワークなし）
- localStorage によるデータ永続化
- テスト: Vitest + jsdom

## データモデル
```
Task {
  id: string          // 一意識別子（crypto.randomUUID()）
  title: string       // タスク名（1文字以上、空白のみ不可）
  completed: boolean  // 完了状態（デフォルト: false）
  createdAt: string   // 作成日時（ISO 8601）
}
```

## 受け入れ基準（EARS記法）

### タスク追加
- When ユーザーがタスク名を入力してEnterキーを押す, the system shall 新しいタスクを未完了状態でリストに追加する
- When ユーザーが追加ボタンをクリックする, the system shall 新しいタスクを未完了状態でリストに追加する
- When ユーザーが空文字または空白のみを入力して追加を試みる, the system shall タスクを追加せず入力を無視する
- When タスクが追加される, the system shall 入力フィールドをクリアする

### タスク完了
- When ユーザーがタスクのチェックボックスをクリックする, the system shall タスクの完了状態をトグルする
- When タスクが完了状態になる, the system shall タスク名に取り消し線を表示する

### タスク削除
- When ユーザーがタスクの削除ボタンをクリックする, the system shall そのタスクをリストから削除する

### タスク編集
- When ユーザーがタスク名をダブルクリックする, the system shall タスク名を編集可能な入力フィールドに変更する
- When ユーザーが編集中にEnterキーを押す, the system shall 変更を保存して表示モードに戻る
- When ユーザーが編集中にEscキーを押す, the system shall 変更を破棄して表示モードに戻る
- When ユーザーが空文字で編集を確定する, the system shall 変更を破棄して元のタスク名を維持する

### フィルタリング
- When ユーザーが「すべて」フィルタを選択する, the system shall すべてのタスクを表示する
- When ユーザーが「未完了」フィルタを選択する, the system shall 未完了のタスクのみ表示する
- When ユーザーが「完了済み」フィルタを選択する, the system shall 完了済みのタスクのみ表示する
- While フィルタが適用されている, the system shall 選択中のフィルタボタンを視覚的に区別する

### 残りタスク数
- While タスクが存在する, the system shall 未完了タスクの件数を「X件の未完了タスク」形式で表示する
- While タスクが0件の, the system shall 「0件の未完了タスク」と表示する

### データ永続化
- When タスクの状態が変更される（追加・完了・削除・編集）, the system shall 即座にlocalStorageにデータを保存する
- When アプリケーションが読み込まれる, the system shall localStorageからタスクデータを復元する

### レスポンシブデザイン
- While 画面幅が768px以上の, the system shall デスクトップ向けレイアウトで表示する
- While 画面幅が768px未満の, the system shall モバイル向けレイアウトに切り替える

## 非機能要件
- 外部依存なし（CDN含む）で動作すること
- ページ読み込み後、即座に操作可能であること

## 依存関係
なし（スタンドアロンアプリケーション）
