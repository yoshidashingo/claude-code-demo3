# プロダクト要求定義書

## プロダクトビジョン

シンプルで直感的なタスク管理アプリケーション。個人のタスクを効率的に管理し、生産性を向上させる。

## ターゲットユーザー

- 個人でタスクを管理したいユーザー
- シンプルなUIを好むユーザー
- ブラウザ上で手軽にタスク管理したいユーザー

## 課題・ニーズ

- タスクの追加・完了・削除を素早く行いたい
- タスクの優先度を視覚的に把握したい
- フィルタリングでタスクを絞り込みたい

## 主要な機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | タスク追加 | タイトルを入力してタスクを作成 |
| 2 | タスク完了 | チェックボックスでタスクを完了状態にする |
| 3 | タスク削除 | 不要なタスクを削除する |
| 4 | タスク編集 | タスクのタイトルをインラインで編集 |
| 5 | フィルタリング | すべて/未完了/完了済みでフィルタリング |
| 6 | タスク数表示 | 残りのタスク数を表示 |
| 7 | データ永続化 | localStorage でデータを保持 |

## 成功の定義

- ユーザーがタスクの追加・完了・削除を3クリック以内で実行できる
- ブラウザをリロードしてもデータが保持される
- レスポンシブデザインでモバイルでも利用可能

## ユーザーストーリー

1. ユーザーとして、新しいタスクを追加したい（素早くタスクを記録するため）
2. ユーザーとして、タスクを完了にしたい（進捗を管理するため）
3. ユーザーとして、タスクを削除したい（不要なタスクを整理するため）
4. ユーザーとして、タスクを編集したい（誤入力を修正するため）
5. ユーザーとして、タスクをフィルタリングしたい（必要なタスクに集中するため）

## 受け入れ条件

- [ ] タスクの追加ができること
- [ ] タスクの完了/未完了の切り替えができること
- [ ] タスクの削除ができること
- [ ] タスクのインライン編集ができること
- [ ] フィルタリング（すべて/未完了/完了済み）ができること
- [ ] 残りタスク数が表示されること
- [ ] localStorage にデータが保存されること
- [ ] レスポンシブデザインであること

## 機能要件

### FR-001: タスク追加
- テキスト入力フィールドにタイトルを入力
- Enterキーまたは追加ボタンでタスクを作成
- 空白のみのタイトルは追加不可

### FR-002: タスク完了
- チェックボックスをクリックで完了/未完了を切り替え
- 完了タスクは取り消し線で表示

### FR-003: タスク削除
- 削除ボタンでタスクを即座に削除

### FR-004: タスク編集
- タスクタイトルをダブルクリックで編集モードに切り替え
- EnterキーまたはBlurで編集確定
- Escキーで編集キャンセル

### FR-005: フィルタリング
- 「すべて」「未完了」「完了済み」の3つのフィルター
- 選択中のフィルターがハイライト表示

### FR-006: タスク数表示
- 未完了のタスク数をフッターに表示

### FR-007: データ永続化
- タスクの追加・更新・削除時に localStorage へ保存
- アプリ起動時に localStorage からデータを復元

## 非機能要件

- **パフォーマンス**: 100件のタスクでもスムーズに動作
- **アクセシビリティ**: キーボード操作に対応
- **ブラウザ対応**: Chrome, Firefox, Safari, Edge の最新版
- **レスポンシブ**: 320px〜1920px の画面幅に対応
