# プロダクト要求定義書

## 1. プロダクトビジョン

### 1.1 プロダクト名
**おいしいマルシェ** - こだわりの食品・グルメ専門ECサイト

### 1.2 ビジョン
全国各地のこだわり食品を、手軽に探して購入できるオンラインマルシェを提供する。

### 1.3 ミッション
- 生産者のこだわり食品を消費者に届ける
- 直感的なUIで食品選びを楽しい体験にする
- カテゴリ別のブラウジングで新しい食との出会いを創出する

## 2. ターゲットユーザー

### 2.1 プライマリユーザー
- **年齢層:** 20〜50代
- **特徴:** 食にこだわりがあり、オンラインショッピングに慣れている
- **ニーズ:** 品質の高い食品を手軽に見つけて購入したい

### 2.2 ユーザーの課題
- スーパーでは手に入らないこだわり食品を探すのが大変
- 食品の詳細情報（原材料、産地など）を事前に確認したい
- 気になった商品をあとでまとめて確認・購入したい

## 3. 主要機能一覧（MVP）

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 商品一覧表示 | カード形式で商品を一覧表示。カテゴリでのフィルタリングが可能 |
| 2 | 商品詳細表示 | 商品の詳細情報（名前、価格、説明、画像、カテゴリ、在庫状況）を表示 |
| 3 | カート機能 | 商品の追加・削除、数量変更、合計金額の表示 |
| 4 | お気に入り機能 | 商品のお気に入り登録・解除、お気に入り一覧の表示 |

## 4. ユーザーストーリー

### US-1: 商品一覧の閲覧
**ユーザーとして、** 商品一覧をカード形式で閲覧したい。
**なぜなら、** 取り扱い商品を視覚的に把握し、興味のある商品を素早く見つけたいから。

**受け入れ条件:**
- [ ] 商品がカード形式（画像・名前・価格・カテゴリ）で一覧表示される
- [ ] カテゴリ別にフィルタリングできる
- [ ] 「全て」を選択すると全商品が表示される
- [ ] レスポンシブデザインで表示される（モバイル: 1列、タブレット: 2列、デスクトップ: 3〜4列）

### US-2: 商品詳細の確認
**ユーザーとして、** 商品の詳細情報を確認したい。
**なぜなら、** 購入前に商品の詳細（説明、価格、在庫状況など）を知りたいから。

**受け入れ条件:**
- [ ] 商品画像が大きく表示される
- [ ] 商品名、価格、説明文、カテゴリが表示される
- [ ] 在庫状況（在庫あり/在庫なし）が表示される
- [ ] 「カートに追加」ボタンが表示される（在庫なしの場合は非活性）
- [ ] 「お気に入りに追加/解除」ボタンが表示される
- [ ] 商品一覧に戻るリンクがある

### US-3: カートの操作
**ユーザーとして、** カートに商品を追加し、カート内の商品を管理したい。
**なぜなら、** 購入したい商品をまとめて管理し、合計金額を確認したいから。

**受け入れ条件:**
- [ ] 商品詳細画面から「カートに追加」ボタンで商品を追加できる
- [ ] カート内の商品の数量を変更（増減）できる
- [ ] カートから商品を削除できる
- [ ] カート内商品の小計と合計金額が表示される
- [ ] カートが空の場合、空であることを示すメッセージが表示される
- [ ] ヘッダーにカート内の商品数がバッジで表示される

### US-4: お気に入りの管理
**ユーザーとして、** 気になる商品をお気に入りに登録したい。
**なぜなら、** あとで検討・購入するためにブックマークしておきたいから。

**受け入れ条件:**
- [ ] 商品詳細画面から「お気に入りに追加/解除」ができる
- [ ] 商品一覧のカードからお気に入りの追加/解除ができる
- [ ] お気に入り一覧ページで登録した商品を確認できる
- [ ] お気に入りが空の場合、空であることを示すメッセージが表示される
- [ ] ヘッダーにお気に入りの件数がバッジで表示される

## 5. 機能要件

### FR-1: 商品データ
- 商品はモックデータとして定義する（バックエンドAPI連携は将来対応）
- 商品データは以下を含む: ID、商品名、説明文、価格、画像URL、カテゴリ、在庫数

### FR-2: カテゴリ
以下のカテゴリを用意する:
- 和菓子
- 洋菓子
- 調味料
- 飲料
- 加工食品

### FR-3: カート状態管理
- カートの状態はアプリケーション内で管理する（Context API）
- ページ遷移してもカートの状態は保持される
- 同一商品の追加時は数量を加算する

### FR-4: お気に入り状態管理
- お気に入りの状態はアプリケーション内で管理する（Context API）
- ページ遷移してもお気に入りの状態は保持される
- お気に入りはトグル操作（追加/解除）

## 6. 非機能要件

### NFR-1: パフォーマンス
- ページ読み込み時間: 3秒以内
- UI操作のレスポンス: 100ms以内

### NFR-2: レスポンシブデザイン
- モバイル（〜640px）、タブレット（641〜1024px）、デスクトップ（1025px〜）に対応
- モバイルファーストのアプローチ

### NFR-3: アクセシビリティ
- セマンティックHTML要素の使用
- 画像にはalt属性を設定
- キーボード操作に対応

### NFR-4: ブラウザ対応
- Chrome、Firefox、Safari、Edgeの最新版に対応

## 7. 制約事項

- フロントエンドのみで完結（バックエンドAPIなし）
- 決済機能は含まない（MVPスコープ外）
- ユーザー認証は含まない（MVPスコープ外）
- データの永続化（localStorage）は将来対応
- 商品データはモックデータを使用

## 8. 成功の定義

- 全4画面（商品一覧、商品詳細、カート、お気に入り）が正常に表示・遷移できる
- カート操作（追加・削除・数量変更）が正しく動作する
- お気に入り操作（追加・解除）が正しく動作する
- レスポンシブデザインが3ブレークポイントで正しく表示される
- ESLintエラーがゼロ
- TypeScript型チェックエラーがゼロ
